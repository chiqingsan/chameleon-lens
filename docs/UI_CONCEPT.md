# UI 概念方向

## 方向

项目后续只聚焦 `ESP + 雷达`。UI 不再承载辅助瞄准、复杂战斗功能或杂项工具，控制面板只保留目标感知相关设置。

## 设计关键词

- 简约：减少控件数量，只展示常用开关和关键数值。
- 高级：深蓝黑背景、低饱和青绿色状态，不使用夸张发光和高饱和渐变。
- 低干扰：覆盖层预览只显示目标点、距离、少量连线和雷达。
- 可扩展：菜单结构按 `ESP / 雷达 / 外观 / 快捷键 / 调试` 五个页签组织，雷达负责屏幕外和背后目标感知，快捷键只承载全局开关。

## 演进记录

- 第一版：竖向面板 + 覆盖层预览，信息层级比较直接，但横向空间利用不足。
- 第二版：横向矩形面板，窗口比例更接近实际悬浮工具，但视觉层次仍偏单调。
- 第三版：参考深色工具台风格，加入左侧导航、状态胶囊和更突出的雷达区域，但仪表盘感偏强。
- 第四版：回到顶部 Tab 结构，配色收敛到深蓝黑 + 青绿色，减少过重的视觉装饰。
- 第五版：收敛为精简仪器面板，尺寸约 900 x 520，保留顶部 Tab、自绘连接状态胶囊、动画开关、ESP 两列开关区和右侧 ESP 效果预览。

当前 PyQt 菜单已按 V5 方向落地：保持横向矩形和顶部 Tab，但避免仪表盘化。ESP 页保持两列开关布局和简洁目标预览；雷达页显示雷达参数和开放 HUD 方位预览，仅保留角标、雷达环和目标点，不再绘制横竖底线、底部基线或内框堆叠。外观页按“绘制参数 / 颜色方案”分区处理透明度、圆点半径和横向色板，外观预览与 ESP 页目标视图保持同一视觉语言，并提供恢复默认外观。快捷键页提供菜单显隐、覆盖层总开关、ESP 绘制和雷达面板开关的按键录入，默认只有菜单显隐为 `F1`。控制项标题和说明采用紧凑两行布局，避免说明文字与标题被拉开。顶部 Tab 不使用底部横条，主容器、标题栏、内容面板、Logo、关键按钮、关闭按钮、状态胶囊、开关、滑条、下拉框、调色板候选色和颜色预览优先采用自绘抗锯齿，避免 Qt StyleSheet 圆角边框毛刺。自绘按钮和页签保留轻量 hover 过渡，滑条在 hover/拖动时给出反馈。目标进程未出现时，覆盖层保持透明安静，不再在屏幕上绘制等待提示卡。菜单仅标题栏可拖动，右上角关闭按钮用于退出程序。
